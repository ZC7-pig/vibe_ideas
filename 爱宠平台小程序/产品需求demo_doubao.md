# 宠物生活服务平台小程序 - 初版Demo产品设计文档（最终版）

## 1. 产品概述

### 1.1 产品定位
聚焦宠物生活垂直领域的本地服务平台，通过数字化手段解决宠物丢失寻回、寄养配种、流浪动物收养及本地宠物服务信息查询等核心需求，构建连接养宠人士与宠物服务的社区化平台。

### 1.2 目标用户
- 城市养猫/狗人群（核心用户）
- 宠物丢失者与发现者
- 有寄养/配种需求的养宠家庭
- 关注流浪动物救助的爱心人士
- 需要查询本地宠物服务的用户

### 1.3 核心价值
- 高效匹配：通过规则匹配提升宠物寻回效率
- 本地连接：聚焦同城服务，解决时空限制
- 安全便捷：提供可信的宠物相关服务信息
- 社区互助：构建养宠人士互助交流生态

## 2. 核心功能模块设计

### 2.1 宠物寻失模块
#### 2.1.1 功能描述
- 发布寻宠信息：用户上传丢失宠物照片，自由输入关键信息（品种、毛色描述、年龄、丢失地点、时间等）
- 发布发现信息：用户上传发现的流浪宠物照片，自由输入发现地点、品种描述、毛色特征等信息
- 规则匹配机制：系统通过**文本关键词（品种描述）+ 毛色关键词 + 地理位置（距离范围）** 三个核心维度进行自动匹配
- 时间范围限定：优先匹配近期（7天内）发布的相关信息
- 双向消息提醒：向匹配度符合条件的寻宠用户与发现用户发送双向提醒通知
- 信息管理：编辑、删除、标记寻回状态

#### 2.1.2 匹配规则细节
- 品种匹配：基于用户输入文本的关键词模糊匹配（如"金毛"可匹配"金毛犬"、"金色毛发的狗"等）
- 毛色匹配：基于用户输入的毛色描述关键词匹配（如"黄色"可匹配"浅黄色"、"黄白相间"等）
- 地理位置：默认匹配5公里范围内的信息，支持用户手动调整范围（3-20公里）
- 时间优先级：按发布时间排序，最新发布的信息优先匹配

#### 2.1.3 用户流程
- 寻宠流程：发起寻宠 → 填写自由文本信息 → 上传照片 → 系统自动匹配 → 接收匹配提醒 → 沟通确认
- 发现流程：发现宠物 → 上传照片 → 填写自由文本信息 → 系统自动匹配 → 接收匹配提醒 → 沟通确认

### 2.2 寄养与配种模块
#### 2.2.1 功能描述
- 服务分类：寄养服务/配种服务两大分类入口
- 信息发布：用户自由发布需求（无需审核），内容包括宠物描述、服务时间、地点、要求等
- 筛选功能：按发布时间、距离范围筛选信息
- 沟通工具：内置聊天功能，支持用户间直接沟通
- 基础评价：服务完成后可进行简单评分（1-5星）

#### 2.2.2 核心交互
- 需求发布表单：简洁输入框设计，支持自由文本描述
- 列表展示：卡片式展示服务信息，包含关键信息与联系方式入口
- 发布流程：填写信息 → 预览 → 直接发布（无审核环节）

### 2.3 流浪动物收养模块
#### 2.3.1 功能描述
- 待领养列表：展示本地可领养的流浪猫狗信息（照片、健康状况、性格描述）
- 领养申请：用户提交领养申请，包含个人基本信息与养宠条件
- 救助站信息：展示合作救助站联系方式（电话、地址）
- 领养指南：提供基础领养流程与注意事项说明

#### 2.3.2 流程设计
- 浏览待领养宠物 → 查看详情 → 提交领养申请 → 救助站线下对接 → 领养成功

### 2.4 本地宠物服务模块（初版简化）
#### 2.4.1 功能描述
- 服务分类入口：仅展示三大分类入口图标（宠物活动、宠物医院、宠物殡葬）
- 功能提示：点击入口后显示"该功能即将上线，敬请期待"提示
- 基础说明：每个入口附带简短文字说明该分类包含的服务类型

#### 2.4.2 页面设计
- 分类图标页：大尺寸图标展示，清晰分类
- 点击反馈：简单动画反馈，明确提示功能状态

## 3. 信息架构

### 3.1 核心实体
- 用户：基本信息、联系方式、地理位置
- 宠物信息：自由文本描述（品种、年龄、性别、毛色）、照片、相关时间地点
- 帖子：寻宠帖、发现帖、寄养帖、配种帖、领养帖（均包含用户关联信息）
- 服务入口：分类标识、说明文字

### 3.2 关系模型
- 用户与帖子：一对多关系（用户可发布多个帖子）
- 帖子与宠物信息：一对一关联
- 用户与用户：通过消息系统建立临时沟通关系

## 4. UI/UX设计要点

### 4.1 页面结构
1. 首页：四大功能模块入口（寻失、寄养配种、收养、本地服务）+ 快捷发布按钮
2. 寻失模块页：寻宠/发现切换入口 + 信息列表 + 发布按钮
3. 寄养配种页：服务类型切换 + 信息列表 + 发布按钮
4. 收养模块页：待领养列表 + 救助站信息入口
5. 本地服务页：三大服务分类入口（仅展示）
6. 个人中心：我的发布、消息中心、设置

### 4.2 交互设计原则
- 核心功能一键可达（发布按钮固定在首页底部）
- 地理位置：自动获取，支持手动修改
- 图片上传：支持拍照/相册选择，简化操作步骤
- 消息通知：匹配提醒（高优先级）、沟通消息（中优先级）

### 4.3 视觉风格
- 主色调：温暖橙色（主色）+ 清新蓝色（辅助色）
- 图标风格：线性简约风格，融入宠物轮廓元素
- 布局特点：大按钮、清晰分区、突出图片展示区域

## 5. 技术实现要点

### 5.1 前端技术
- 框架：微信小程序原生框架
- 地图：集成微信地图SDK，实现定位与距离计算
- 图片处理：客户端压缩 + 预览功能
- 表单设计：以自由输入框为主，减少选择类控件

### 5.2 后端技术
- 服务架构：RESTful API设计
- 数据库：用户数据、帖子信息、宠物描述存储
- 缓存策略：地理位置数据、近期帖子缓存

### 5.3 匹配功能实现
- 规则引擎：基于文本关键词匹配（品种、毛色）+ 地理位置计算
- 关键词提取：简单字符串匹配算法（初版）
- 匹配触发：新帖发布时即时触发匹配检查
- 通知机制：微信小程序模板消息推送

### 5.4 数据安全
- 用户信息加密存储
- 地理位置信息脱敏处理
- 基础权限控制：区分普通用户与管理员

## 6. 初版Demo范围
- 核心功能：宠物寻失模块（含规则匹配）、寄养配种信息发布与浏览、流浪动物收养列表
- 基础功能：用户注册登录、简单聊天沟通、个人中心基础功能
- 占位功能：本地宠物服务模块（仅展示入口）
- 不含功能：支付系统、复杂评价体系、社区社交功能、审核机制

## 7. 后续迭代方向
- 优化匹配算法：引入更智能的文本分析与特征提取
- 完善本地服务：接入实际服务信息与详情展示
- 增加社交元素：评论、点赞、分享功能
- 引入信用体系：基于用户行为建立简单信用评分
- 开发工具类功能：宠物年龄计算、疫苗提醒等
